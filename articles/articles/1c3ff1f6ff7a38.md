# おみくじアプリ開発：データ設計をブラッシュアップしてアプリに代入。

## 今日やったこと

* おみくじアプリのデータ設計を定義
* 各運勢ごとのパターンを整理（18通り以上確保）
* 「今日のひとこと（comment）」を追加

## データ構造について

通常のおみくじデータに加えて、オリジナリティとして「今日のひとこと」を追加しました。

**一例** 

```js
{
    fortune: "大吉",
    patterns: [
        {
            wish: "今まで積み重ねてきた努力が報われる",
            waitPeople: "知らせが舞い込む",
            health: "体調良好、無理しなければ安定",
            work: "周囲から頼られることが多くなりそう",
            study: "普段読まない分野の本を読むと意外にもハマってしまいそう",
            trip: "地元の店に立ち寄ってみよう",
            message: "小さな一歩が、大きな流れをつくる日。"
        }
    ]
}
```

## 気づき

* データを増やすと、それに対応するHTML側の要素も必要になる
* 今回は `message` を追加したため、表示用のタグも新しく用意した
* データ設計とUIはセットで考える必要があると実感

## 学び

* JSONライクな構造でデータを持つと、拡張がしやすい
* 「ひとこと」を追加するだけで、アプリの体験が少し華やかになる
* 小さな改善でもユーザー体験に影響する

## 次やること
