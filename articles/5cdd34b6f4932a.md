この記録では、GitHubとGitのインストール方法は省くので、ご了承ください。

##　GitとGitHubの違いについて

###　Gitとは？

Gitは、ローカル上でデータを記録するツールです。

例えるなら、**自分の頭の中。**

なので、インターネットの先にいる、「だれか」に**制作したコードが共有されることはありません。**

- 「ポートフォリオとして公開したい」

- 「インターネットを通じて誰かに共有したい」

このような場合は、次に解説するGitHubを用います。

###　GitHubとは？

先に記載をしましたが、GitHubとは

**インターネット上に公開**された誰もが確認できる開かれた空間となります。

まさに、ブログのような形で全国のあらゆる人に共有ができるのです。

##　GitHubに制作したコードを置くには？

まず初めに、GitHubに登録したいプロジェクトフォルダをエクスプローラーで開き、**GitBush**を起動しましょう。

プロジェクトフォルダと、Gitを始めに連携します。

**git init**

その後、以下3つのコードを用いるとGitHubにファイルデータを落とし込むことができます。

- git add .
- git commit -m "変更内容"
- git push

以下にそれぞれのコードの意味を簡潔に解説します。

###　git add .

**「変更したファイルを記録対象にする」**

このコードを書く際、「add」の直後に半角スペースを入れる点に注意してください。

###　git commit -m "変更内容"

**「この作業を履歴として保存」**

「-m」の直後に、更新したデータの概要を記載しないとエラーになります。

後から見返した際の履歴としても見やすいので、**"〇〇を実装"** 等　記載しましょう。

ちなみに、コメントを書かない場合は **git commit**のみで登録できます。（おそらく）

###　git push

**「PCの履歴をGitHubにアップロード」**

ついに、ローカルデータをGitHubと接続します。

ただし、初めてGitHubにアップロードする場合は**git push**ではなく、以下のコードを記載します。

**git push -u origin main**

もし、GitHub上に既に何らかのデータが入っており（例. ReadMe）、そのデータがローカル環境にない場合、以下のエラーが表示される場合があります。

**[rejected] main -> main (fetch first)**
先にGitHubの内容を取得してから送ってください。

**remote contains work that you do not have locally**
GitHubにあるファイルを、あなたのPCがまだ持っていません。

このようなエラーメッセージが表示された場合、
ローカル環境に入っていないGitHubのみに入っているデータを取り込む必要性があります。

以下のコードを、**GitBush**で実行しましょう。

**git pull origin main --allow-unrelated-histories**
PC側に入っていない、GitHubのみに入っているデータを取り込む。

実行すると、以下のようなメッセージが表示される場合があります。

Merge branch 'main' of https://github.com/...
 Please enter a commit message
GitHubの内容とPCの内容を合体（merge）するのでコメントを書いてください。

以下の対処を行いましょう。

- **Esc**した後に、**:wq**と入力し、**Enter**してください。

上記対応は、特にコメントを書かずにそのまま同期して終了という意味になります。

以降、今後更新を行う際は以下の手順を**GitBush**で実行することでローカル上のデータとGitHubを連携できます。

1. git add .
2. git commit -m "変更内容"
3. git push
