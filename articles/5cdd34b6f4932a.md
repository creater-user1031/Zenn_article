この記録では、GitHubとGitのインストール方法は省くので、ご了承ください。

## GitとGitHubの違いについて

### Gitとは？

Gitは、ファイルの変更履歴を管理するツールです。

基本的にはPC（ローカル環境）で動作し、
- いつ
- どのファイルを
- どのように変更したのか

といった履歴を記録していきます。

例えるなら、**作業日記**のようなものです。

ただし、Git単体ではPCの中で履歴を管理するだけなので、  
インターネット上の他の人と共有することはできません。

### GitHubとは？

GitHubとは

**Gitのデータをインターネット上に保存できるサービス**です。

GitはPCの中で履歴を管理するツールですが、
GitHubを使うことで

- コードをインターネット上に保存する
- 他の人と共有する
- チームで共同開発する

といったことが可能になります。

**まとめ**

- Gitは「履歴を管理するツール」  
- GitHubは「その履歴をネット上に保存する場所」

## GitHubに制作したコードを置くには？

まず初めに、GitHubに登録したいプロジェクトフォルダをエクスプローラーで開き、**GitBash**を起動しましょう。

### ①Gitを開始する

**git init**

このコマンドは

**「このフォルダでGitを使います」**

という宣言になります。

### ②GitHubと接続する

**git remote add origin https://github.com/ユーザー名/zenn-content.git**


このコマンドは

**「このGitHubの場所にデータを送ります」**

という接続設定になります。

ここを設定していない場合、次のエラーが出ます。

**fatal: No configured push destination**


これは

**GitHubの送り先URLが設定されていない**

という意味になります。

### ③ GitHubにアップロードする

**git add .**

**「変更したファイルを記録対象にする」**

という意味です。

このコードを書く際、「add」の直後に半角スペースを入れる点に注意してください。

### ④gitの情報を更新

**git commit -m "変更内容"**

**変更内容を履歴として保存する**

という意味です。

`-m` は **message（コメント）** の意味です。

「-m」の直後に、更新したデータの概要を記載しないとエラーになります。

後から見返した際に履歴の内容を瞬時に判断できるように、**"〇〇を実装"** 等　記載しましょう。

### ⑤main branchの作成

**git branch -M main**

このコマンドは、現在のブランチ名を main に変更するコマンドです。

Gitでは作業履歴を**ブランチ**という単位で管理します。
GitHubでは、基本的に mainブランチ が標準のブランチとして扱われます。

そのため、初回のアップロード前にブランチ名を main に設定しておきます。

### git push

**「PCの履歴をGitHubにアップロード」**

という意味になります。

さて、ローカルデータをGitHubと接続します。

ただし、初めてGitHubにアップロードする場合は**git push**ではなく、以下のコードを記載します。

**git push -u origin main**

これは

- origin → GitHubの場所  
- main → ブランチ名  

を指定してアップロードするコマンドです。

もし、GitHub上に既に何らかのデータが入っており（例. ReadMe）、そのデータがローカル環境にない場合、以下のエラーが表示される場合があります。

**[rejected] main -> main (fetch first)**
先にGitHubの内容を取得してから送ってください。

**remote contains work that you do not have locally**
GitHubにあるファイルを、あなたのPCがまだ持っていません。

このようなエラーメッセージが表示された場合、
ローカル環境に入っていないGitHubのみに入っているデータを取り込む必要性があります。

以下のコードを、**GitBush**で実行しましょう。

**git pull origin main --allow-unrelated-histories**
PC側に入っていない、GitHubのみに入っているデータを取り込む。

実行すると、以下のようなメッセージが表示される場合があります。

Merge branch 'main' of https://github.com/...
 Please enter a commit message
GitHubの内容とPCの内容を合体（merge）するのでコメントを書いてください。

以下の対処を行いましょう。

- **Esc**した後に、**:wq**と入力し、**Enter**してください。

上記対応は、特にコメントを書かずにそのまま同期して終了という意味になります。

以降、今後更新を行う際は以下の手順を**GitBush**で実行することでローカル上のデータとGitHubを連携できます。

1. git add .
2. git commit -m "変更内容"
3. git push
